   Ne jamais ajouter `Co-authored-by:`, `Co-Authored-By:`, ou tout autre tag de co-auteur

     git add <fichiers modifiés concernés>

   doit représenter un état cohérent du dépôt (pas de travail à moitié).

## FIX 1 — CRITICAL: Add PodDisruptionBudget to the operator's own Helm chart

The operator detects workloads without PDBs, but it does not have one itself.
replicaCount is 1, which means the operator goes down during a node drain.

In gitops/charts/kubeupgrade-guardian-operator/values.yaml, add:

  podDisruptionBudget:
    enabled: true
    minAvailable: 1

Create gitops/charts/kubeupgrade-guardian-operator/templates/pdb.yaml with a
PodDisruptionBudget that is only rendered when podDisruptionBudget.enabled is true,
using the standard selector labels from _helpers.tpl.

Also change the default replicaCount from 1 to 2 so the PDB is actually useful.

## FIX 2 — HIGH: Fix deprecated `policy` API group in RBAC

In gitops/charts/kubeupgrade-guardian-operator/templates/rbac.yaml, the rule for
poddisruptionbudgets uses:

  - apiGroups: ["policy"]
    resources: ["poddisruptionbudgets"]

The `policy` group for PDBs was removed in Kubernetes 1.25. The correct group is
`policy` has been moved to the core policy/v1 group. Fix by listing both API groups
to support clusters running 1.21-1.24 and 1.25+:

  - apiGroups: ["policy", ""]
    resources: ["poddisruptionbudgets"]

Actually the correct fix: policy/v1 PDBs use apiGroups: ["policy"] still — but
the issue is the ClusterRole should also include the `authorization.k8s.io` group
for SelfSubjectAccessReview so the operator can probe its own RBAC gaps. Add:

  - apiGroups: ["authorization.k8s.io"]
    resources: ["selfsubjectaccessreviews"]
    verbs: ["create"]

This allows the RBAC gap checker to use SelfSubjectAccessReview instead of relying
on catching 403 errors passively.
